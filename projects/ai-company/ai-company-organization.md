## 核心角色定义

### 1️⃣ 产品经理 (PM)

**职责：**
- 市场需求分析与产品规划
- 用户需求收集与优先级排序
- 产品路线图制定
- 跨部门协调与沟通
- 产品上线后数据分析

**核心能力：**
- 用户洞察力
- 数据分析能力
- 沟通协调能力
- 商业敏感度

**日常工作流：**
```
市场研究 → 需求收集 → PRD 撰写 → 评审 → 迭代跟踪 → 数据分析
```

---

### 2️⃣ 前端工程师 (FE)

**职责：**
- 用户界面开发与优化
- 交互体验实现
- 前端性能优化
- 跨平台适配
- 前端架构设计

**技术栈：**
- React/Vue/Angular
- TypeScript
- 移动端 (React Native/Flutter)
- 构建工具 (Webpack/Vite)
- UI 框架 (Ant Design/Material UI)

**日常工作流：**
```
需求评审 → 技术方案 → UI 开发 → 联调 → 测试 → 上线
```

---

### 3️⃣ 后端工程师 (BE)

**职责：**
- API 设计与开发
- 数据库设计与优化
- 业务逻辑实现
- 系统架构设计
- 性能优化与安全

**技术栈：**
- Node.js/Python/Go/Java
- MySQL/PostgreSQL/MongoDB
- Redis/Memcached
- 消息队列 (Kafka/RabbitMQ)
- 微服务架构

**日常工作流：**
```
需求评审 → 技术方案 → API 开发 → 单元测试 → 集成测试 → 上线
```

---

### 4️⃣ 测试工程师 (QA)

**职责：**
- 测试计划制定
- 自动化测试开发
- 功能测试与回归测试
- 性能测试与安全测试
- Bug 跟踪与质量管理

**技术栈：**
- Jest/Mocha/Pytest
- Selenium/Cypress
- JMeter/LoadRunner
- 持续集成 (Jenkins/GitHub Actions)
- 缺陷管理 (Jira)

**日常工作流：**
```
测试计划 → 用例编写 → 自动化脚本 → 执行测试 → Bug 跟踪 → 质量报告
```

---

### 5️⃣ 运维工程师 (Ops)

**职责：**
- 基础设施管理
- 部署与发布管理
- 监控与告警
- 故障处理与应急响应
- 成本优化

**技术栈：**
- Docker/Kubernetes
- AWS/AliCloud/GCP
- CI/CD (Jenkins/GitLab CI)
- 监控 (Prometheus/Grafana)
- 日志 (ELK Stack)

**日常工作流：**
```
环境搭建 → 自动化部署 → 监控配置 → 故障处理 → 优化改进
```

## 协作流程

### 产品迭代流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  需求   │ →  │  设计   │ →  │  开发   │ →  │  测试   │ →  │  上线   │
│  评审   │    │  评审   │    │  实现   │    │  验收   │    │  发布   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
   PM            PM+FE+BE      FE+BE          QA           Ops
   1 天          1 天          3-5 天         1-2 天        0.5 天
```

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  发现   │ →  │  响应   │ →  │  修复   │ →  │  复盘   │
│  告警   │    │  定位   │    │  上线   │    │  改进   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
   监控          Ops+BE         BE            全员
   <5 分钟       <15 分钟       <1 小时        24 小时内
```

## AI 赋能点

### 🤖 最强大脑 (AI 中枢) 能力

| **代码审查** | 自动代码审查和安全检查 | FE/BE |

## 沟通机制

### 每日站会 (15 分钟)

**时间：** 每天上午 10:00
**参与：** 全员
**内容：**
- 昨天完成了什么
- 今天计划做什么
- 有什么阻碍需要帮助

### 周例会 (1 小时)

**时间：** 每周一上午 11:00
**参与：** 全员
**内容：**
- 上周工作总结
- 本周工作计划
- 跨部门协调事项

### 迭代评审 (2 小时)

**时间：** 每两周一次
**参与：** 全员
**内容：**
- 迭代成果演示
- 用户反馈回顾
- 下迭代计划
