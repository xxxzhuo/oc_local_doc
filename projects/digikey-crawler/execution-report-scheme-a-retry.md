## 二、问题分析

### 失败原因

| 阶段 | 问题 | 原因 |
|------|------|------|
| **步骤 1** | 403 Forbidden | DigiKey 反爬机制 |
| **步骤 2** | 403 Forbidden | 页面访问被封锁 |
| **步骤 3** | 404 Not Found | PDF 链接猜测错误 |

### 根本原因

1. **DigiKey 反爬** - HTTP 请求被识别为爬虫，返回 403
2. **PDF 链接变更** - Raspberry Pi 官网更新了文档结构
3. **链接猜测失败** - 无法准确预测 PDF URL 模式

## 三、经验总结

### 教训

1. ❌ **HTTP 请求不适合** - DigiKey 有较强的反爬机制
2. ❌ **PDF 链接不可猜测** - 需要实际爬取产品页面
3. ✅ **已有数据最可靠** - 之前成功下载的 6 个 PDF 是有效的

### 建议

1. **使用浏览器自动化** - Playwright 可以绕过反爬
2. **使用官方 API** - DigiKey API 或 Raspberry Pi API
3. **手动收集链接** - 从官方网站查找正确的 PDF 链接
